Master Trees from Zero to Interview Level

Trees are hierarchical data structures. Most tree problems are solved using recursion, DFS, BFS, or binary-search-tree rules. This page gives you visuals, traversal animation, C code, patterns, and interview roadmap.

Think of a tree as a linked list that branches. Every node can point to children.

Core Terms

  • Root: top node
  • Parent / child relation
  • Leaf: no child
  • Height: longest depth below node

Main Patterns

  • DFS: recursion / stack
  • BFS: queue / level order
  • BST: left < root < right
  • Divide problem into subtrees

Must Solve

  • Max Depth
  • Invert Tree
  • Validate BST
  • Lowest Common Ancestor

Binary Tree
A tree where each node has at most two children: left and right. It does not require sorted order.
struct Node {
    int data;
    struct Node *left;
    struct Node *right;
};

Binary Search Tree
BST keeps sorted order. Left subtree values are smaller. Right subtree values are greater.
Node* search(Node* root, int key) {
    if (root == NULL || root->data == key)
        return root;

    if (key < root->data)
        return search(root->left, key);

    return search(root->right, key);
}






  
Important Tree Algorithms

  

    
1. DFS Traversal
Go deep first. Used in almost every recursive tree problem.
O(n)

    
2. BFS Traversal
Visit level by level using a queue.
O(n)

    
3. Height / Depth
Return 1 + max height of left and right subtree.
Recursive

    
4. Diameter
Longest path between any two nodes.
Postorder

    
5. Validate BST
Use min/max range to check every node.
Range check

    
6. LCA
Lowest common ancestor of two nodes.
Path logic

    
7. Invert Tree
Swap left and right recursively.
Classic

    
8. Path Sum
Carry remaining target while moving down.
Backtracking

    
9. Build Tree
Construct tree from preorder + inorder.
Advanced

  






  
Core C Templates

  


int height(Node* root) {
    if (root == NULL)
        return 0;

    int lh = height(root->left);
    int rh = height(root->right);

    return 1 + (lh > rh ? lh : rh);
}


int countNodes(Node* root) {
    if (root == NULL)
        return 0;

    return 1 + countNodes(root->left)
             + countNodes(root->right);
}


Node* invertTree(Node* root) {
    if (root == NULL)
        return NULL;

    Node* temp = root->left;
    root->left = root->right;
    root->right = temp;

    invertTree(root->left);
    invertTree(root->right);
    return root;
}


int isLeaf(Node* root) {
    return root != NULL &&
           root->left == NULL &&
           root->right == NULL;
}

Interview trick: If the problem talks about subtree, height, path, ancestor, leaf, or left/right child, think recursion first. If it talks about level, nearest, width, or zigzag, think BFS queue.
